<!-- LaTeX compilation command (from RESEARCH.md Pitfall 5 — latexmk changes to file directory): -->
Compile from repo root: `latexmk -pdf paper/main.tex`
   OR: `cd paper && latexmk -pdf main.tex`
Both work: latexmk changes to the input file's directory before processing.

Common fatal error fixes:
- "Undefined control sequence \doublespacing" → \usepackage{setspace} missing from preamble
- "File not found: ../output/tables/X.tex" → check output/ directory or comment out missing \input{}
- "\begin{table} ended by \end{document}" → table fragment double-wrapped (remove outer \begin{table})
- "Missing \begin{document}" → preamble error, often unclosed brace
- "Undefined citation keys" → BibTeX not yet run; latexmk runs bibtex automatically on second pass

README.md two-command workflow (D-07):
```
pip install -r requirements.txt
python run_all.py
```
Then optionally: `cd paper && latexmk -pdf main.tex`
</interfaces>
